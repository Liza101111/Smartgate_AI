## Database schema

### heatpump_log
Sensor readings logged periodically (every few minutes).

Columns:
- log_id, heatpump_id, log_ulid, TIMESTAMP (datetime)
- outdoor_temp — outdoor air temperature (°C)
- brine_out, brine_in — brine circuit supply/return (°C)
- brine_pressure — brine circuit pressure (bar)
- production_out, production_in — heating circuit supply/return (°C)
- production_pressure — heating circuit pressure (bar)
- buffer_temp — buffer tank temperature (°C)
- heating_power — instantaneous heating power (kW)
- electrical_consumption — instantaneous electrical consumption (kW)
- setpoint_temp — active temperature setpoint (°C)
- compressor_suction_temperature, compressor_suction_pressure
- compressor_discharge_pressure, compressor_discharge_temperature
- superheat, inverter_temperature, evaporation_temperature, condensing_temperature
- maximum_reference_of_compressor
- heating_useful_energy_current_day — cumulative useful heating energy since midnight (kWh); resets at midnight
- energy_consumption_current_day — cumulative electrical energy consumed since midnight (kWh); resets at midnight
- distribution_temp2
- group_1_delivery_temperature, group_2_delivery_temperature, group_3_delivery_temperature, group_4_delivery_temperature, group_5_delivery_temperature
- status

To calculate COP for a period, take MAX of the daily counters per day, then sum:
  SELECT DATE(TIMESTAMP) as day,
         MAX(heating_useful_energy_current_day) as useful,
         MAX(energy_consumption_current_day) as consumed
  FROM heatpump_log
  WHERE heatpump_id = <id> AND TIMESTAMP >= '<start>' AND TIMESTAMP < '<end>'
  GROUP BY DATE(TIMESTAMP)
Then COP = SUM(useful) / SUM(consumed).

Examples of date ranges for COP queries (today is 2026-06-16):
- "last month" (May 2026): TIMESTAMP >= '2026-05-01' AND TIMESTAMP < '2026-06-01'
- "this month" (June 2026 to date): TIMESTAMP >= '2026-06-01' AND TIMESTAMP < '2026-06-17'
- "last year" (2025): TIMESTAMP >= '2025-01-01' AND TIMESTAMP < '2026-01-01'
- "this year" (2026 to date): TIMESTAMP >= '2026-01-01' AND TIMESTAMP < '2026-06-17'

### heatpump_status
Status snapshots logged periodically.

Columns:
- status_id, heatpump_id, log_ulid, timestamp (datetime)
- status (int): 1=OFF, 2=ON
- operation_program (int): 0=Winter, 1=Summer
- alarm_level (int)
- lock_alarm_code, long_time_alarm_code, recurrence_alarm_code
- active_alarm_code_1 through active_alarm_code_5
- reset_recurrence_alarm_bus, regeneration (tinyint)
- buffer_heating_setpoint, g1_heating_setpoint, g2_heating_setpoint, g3_heating_setpoint, g4_heating_setpoint
- G5_HEATING_SETPOINT, G6_HEATING_SETPOINT, G7_HEATING_SETPOINT
- heating_buffer_bus_demand, buffer_tank_setpoint, offset_heating_buffer_tank
- maximum_reference_of_compressor
- regeneration_temp, regeneration_time
- compressor_suction, superheat
- heating_energy_daily, heating_energy_monthly (kWh)
- energy_consumption_daily, energy_consumption_monthly (kWh)

### alarms
Alarm history records.

Columns:
- log_id, heatpump_id, log_ulid, timestamp (datetime)
- alarm_level (tinyint)
- lock_alarm_code, long_time_alarm_code, recurrence_alarm_code
- active_alarm_code_1 through active_alarm_code_5
- reset_recurrence_alarm_bus

To get alarm descriptions, join with or look up ALARM_CODES.

### regeneration_log
Records each regeneration cycle.

Columns:
- id, heatpump_id
- started_at (datetime)
- reason (varchar)

### ALARM_CODES
Lookup table for alarm code descriptions. Alarm codes are manufacturer-specific.

Columns:
- ID (bigint)
- HPUMP_GROUP (int) — manufacturer group, links to HPUMP_CODES.HPUMP_GROUP
- ERROR_CODE (int) — the numeric alarm code
- ERROR_TEXT (varchar) — human-readable description

To translate an alarm code, first find the heatpump's HPUMP_GROUP (from the heatpump or HPUMP_CODES table), then:
  SELECT ERROR_TEXT FROM ALARM_CODES WHERE HPUMP_GROUP = <group> AND ERROR_CODE = <code>
